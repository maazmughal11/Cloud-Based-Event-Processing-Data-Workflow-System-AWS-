# ☁️ Cloud-Based Event Processing System (AWS)

**Author:** Maaz Mughal  

---

## 📌 Overview

This project demonstrates a cloud-based event processing system built using AWS services.  
It simulates real-time events and processes them through a scalable pipeline for storage, transformation, and structured querying.

The focus is on **cloud systems, automation, and end-to-end workflow design**.

---

## 🏗️ Architecture

Kafka → AWS Lambda → Amazon S3 → AWS Glue → Amazon Redshift

---

## 🔧 Key Features

- Event-driven data ingestion using Kafka and Lambda  
- Scalable cloud storage with Amazon S3  
- Data transformation using AWS Glue (PySpark)  
- Structured storage in Amazon Redshift  
- Infrastructure deployment using Terraform  

---

## 🧰 Tech Stack

AWS (Lambda, S3, Glue, Redshift), Kafka, Python, PySpark, Terraform, Docker  

---

## 📂 Project Structure

- `ingestors/` – Event ingestion  
- `etl/` – Data processing  
- `warehouse/` – Redshift schemas  
- `infra/` – Terraform setup  
- `tests/` – Validation tests
